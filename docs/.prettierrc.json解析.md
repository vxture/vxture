# .prettierrc.json 配置文件详解

本文档详细解释了 Vxture 项目中 `.prettierrc.json` 文件的各个部分及其作用，帮助开发者理解 Prettier 代码格式化配置。

## 什么是 .prettierrc.json？

`.prettierrc.json` 是 Prettier 的配置文件，用于定义代码格式化的规则。Prettier 是一个代码格式化工具，它能够根据一组固定的规则自动格式化代码，确保团队中所有成员的代码风格一致。

## 配置文件概览

```json
{
  "semi": true,
  "singleQuote": true,
  "tabWidth": 2,
  "trailingComma": "es5",
  "printWidth": 100,
  "bracketSpacing": true,
  "arrowParens": "avoid",
  "endOfLine": "lf"
}
```

## 配置选项详解

### 基础格式化规则

```json
{
  "semi": true,
  "singleQuote": true,
  "tabWidth": 2,
  "printWidth": 100
}
```

| 选项          | 值     | 说明                   | 示例                                                 |
| ------------- | ------ | ---------------------- | ---------------------------------------------------- |
| `semi`        | `true` | 在语句末尾添加分号     | `const name = 'John';`                               |
| `singleQuote` | `true` | 使用单引号而不是双引号 | `const name = 'John';` 而不是 `const name = "John";` |
| `tabWidth`    | `2`    | 缩进使用两个空格       | 代码缩进为 2 个空格                                  |
| `printWidth`  | `100`  | 行宽限制，超过会换行   | 每行代码最多 100 个字符                              |

### 逗号和括号规则

```json
{
  "trailingComma": "es5",
  "bracketSpacing": true,
  "arrowParens": "avoid"
}
```

| 选项             | 值        | 说明                                | 示例                                       |
| ---------------- | --------- | ----------------------------------- | ------------------------------------------ |
| `trailingComma`  | `"es5"`   | 在ES5中有效的多行结构中添加尾随逗号 | 对象、数组等多行结构的最后一项后添加逗号   |
| `bracketSpacing` | `true`    | 在对象字面量的括号内添加空格        | `{ foo: bar }` 而不是 `{foo: bar}`         |
| `arrowParens`    | `"avoid"` | 尽可能省略箭头函数参数的括号        | 单参数: `x => x * x` 而不是 `(x) => x * x` |

### 文件格式规则

```json
{
  "endOfLine": "lf"
}
```

| 选项        | 值     | 说明                                                              |
| ----------- | ------ | ----------------------------------------------------------------- |
| `endOfLine` | `"lf"` | 使用 LF (Line Feed) 作为行尾字符，适用于 Linux/macOS 和跨平台开发 |

## 代码示例

### 格式化前

```javascript
// 未格式化的代码
const user = { name: 'John', age: 30, isAdmin: false };
function calculateTotal(items, tax) {
  return items.reduce((acc, item) => acc + item.price, 0) * (1 + tax);
}
const greeting = name => {
  return 'Hello, ' + name + '!';
};
```

### 格式化后

```javascript
// 格式化后的代码
const user = { name: 'John', age: 30, isAdmin: false };

function calculateTotal(items, tax) {
  return items.reduce((acc, item) => acc + item.price, 0) * (1 + tax);
}

const greeting = name => {
  return 'Hello, ' + name + '!';
};
```

## 与 ESLint 的集成

Prettier 专注于代码格式化，而 ESLint 专注于代码质量。它们可以很好地配合使用：

1. 在 `.eslintrc.json` 中添加 Prettier 相关配置：

   ```json
   {
     "extends": [
       // 其他扩展...
       "prettier"
     ],
     "plugins": [
       // 其他插件...
       "prettier"
     ],
     "rules": {
       "prettier/prettier": "error"
       // 其他规则...
     }
   }
   ```

2. 安装所需依赖：
   ```bash
   npm install --save-dev eslint-config-prettier eslint-plugin-prettier
   ```

## 与编辑器集成

VS Code 中使用 Prettier：

1. 安装 Prettier 扩展

2. 配置保存时自动格式化：
   ```json
   // .vscode/settings.json
   {
     "editor.defaultFormatter": "esbenp.prettier-vscode",
     "editor.formatOnSave": true
   }
   ```

## 最佳实践

1. **与团队保持一致**：Prettier 配置应该在团队成员之间保持一致，避免不必要的格式变更

2. **使用 `.prettierignore`**：创建 `.prettierignore` 文件排除不需要格式化的文件：

   ```
   # 构建输出
   build/
   dist/

   # 生成的文件
   .next/

   # 第三方代码
   node_modules/

   # 其他不需要格式化的文件
   *.min.js
   ```

3. **将格式化集成到 CI 流程**：在持续集成中添加格式检查步骤：

   ```json
   // package.json
   {
     "scripts": {
       "format:check": "prettier --check '**/*.{js,ts,tsx,json,md,scss}'"
     }
   }
   ```

4. **与其他工具配合**：确保 Prettier 配置不会与 ESLint、TypeScript 等工具冲突

5. **文档化特殊决策**：如果团队选择了与常规不同的格式化规则，记录原因和考虑因素

## 常见问题解决

1. **格式化冲突**：如果发现 ESLint 和 Prettier 冲突，确保：
   - `eslint-config-prettier` 正确配置
   - `prettier` 是 `extends` 数组的最后一项

2. **部分文件不格式化**：检查：
   - `.prettierignore` 文件是否排除了这些文件
   - 文件类型是否受支持
   - 编辑器扩展是否正确配置

3. **自定义特定文件类型的格式化**：使用 `overrides` 配置：
   ```json
   {
     "overrides": [
       {
         "files": "*.md",
         "options": {
           "tabWidth": 4
         }
       }
     ]
   }
   ```
