# .eslintrc.json Configuration Explained

This document explains the structure and options of the `.eslintrc.json` file in the Vxture project, focusing on a modern Next.js + TypeScript monorepo setup and best practices for linting and code quality.

## What is .eslintrc.json?

`.eslintrc.json` is the configuration file for ESLint, a static code analysis tool for JavaScript and TypeScript. It defines code quality, style, and best practice rules to ensure consistency and catch errors early in development.

## Example File Structure

```json
{
  "root": true,
  "parser": "@typescript-eslint/parser",
  "parserOptions": {
    "ecmaVersion": 2022,
    "sourceType": "module",
    "ecmaFeatures": { "jsx": true },
    "project": "./tsconfig.json"
  },
  "extends": [
    "next/core-web-vitals",
    "plugin:react/recommended",
    "plugin:jsx-a11y/recommended",
    "plugin:import/errors",
    "plugin:import/warnings",
    "plugin:import/typescript",
    "plugin:@typescript-eslint/recommended",
    "plugin:security-node/recommended",
    "prettier"
  ],
  "plugins": [
    "react",
    "jsx-a11y",
    "import",
    "@typescript-eslint",
    "sonarjs",
    "react-perf",
    "prettier",
    "unused-imports",
    "security-node"
  ],
  "env": {
    "browser": true,
    "node": true,
    "es2022": true
  },
  "rules": {
    /* ...see below... */
  },
  "settings": {
    /* ...see below... */
  },
  "overrides": [
    /* ...see below... */
  ]
}
```

## Key Configuration Options

### Base Settings

- **root**: Ensures this config is the root and ESLint will not look further up the directory tree.
- **parser**: Uses `@typescript-eslint/parser` to enable TypeScript support.
- **parserOptions**: Sets ECMAScript version, module type, JSX support, and points to the TypeScript config for type-aware linting.

### Extends

The `extends` array applies recommended rule sets in order (later entries override earlier ones):

- `next/core-web-vitals`: Next.js and web performance best practices.
- `plugin:react/recommended`: React best practices.
- `plugin:jsx-a11y/recommended`: Accessibility rules for JSX.
- `plugin:import/errors`, `plugin:import/warnings`, `plugin:import/typescript`: Import/export correctness and TypeScript support.
- `plugin:@typescript-eslint/recommended`: TypeScript-specific rules.
- `plugin:security-node/recommended`: Node.js security checks.
- `prettier`: Disables rules that conflict with Prettier formatting.

### Plugins

The `plugins` array enables additional rule sets:

- `react`, `jsx-a11y`, `import`, `@typescript-eslint`, `sonarjs`, `react-perf`, `prettier`, `unused-imports`, `security-node`

### Environment

Defines global variables and syntax features:

- `browser`: Enables browser globals (e.g., `window`, `document`).
- `node`: Enables Node.js globals.
- `es2022`: Enables ES2022 features.

### Rules

Rules are grouped by category for clarity. Example highlights:

#### React Rules

- `react/react-in-jsx-scope`: Off (not needed in React 17+)
- `react/prop-types`: Off (TypeScript handles types)
- `react-hooks/rules-of-hooks`: Error (enforces correct hook usage)
- `react-hooks/exhaustive-deps`: Warn (checks hook dependencies)
- `react/jsx-key`: Error (requires `key` in lists)
- `react/no-unescaped-entities`: Warn
- `react/display-name`: Off

#### Accessibility (a11y) Rules

- `jsx-a11y/anchor-is-valid`: Warn
- `jsx-a11y/alt-text`: Error
- `jsx-a11y/img-redundant-alt`: Error
- `jsx-a11y/aria-role`: Error

#### TypeScript Rules

- `@typescript-eslint/no-unused-vars`: Off (handled by unused-imports)
- `@typescript-eslint/no-explicit-any`: Warn
- `@typescript-eslint/explicit-function-return-type`: Off
- `@typescript-eslint/explicit-module-boundary-types`: Off
- `@typescript-eslint/no-non-null-assertion`: Warn

#### Import Rules

- `import/order`: Enforces import grouping, newlines, and alphabetization.
- `import/no-duplicates`: Error
- `import/no-unresolved`: Off (TypeScript handles this)

#### Code Quality & Security

- `no-console`: Warn (allows `warn`, `error`, `info`)
- `no-debugger`: Warn
- `sonarjs/cognitive-complexity`: Warn if function complexity > 15
- `sonarjs/no-duplicate-string`: Warn if string repeated > 3 times
- `sonarjs/no-identical-functions`: Warn
- `security-node/detect-insecure-randomness`: Warn
- `react-perf/jsx-no-new-object-as-prop`: Warn
- `react-perf/jsx-no-new-array-as-prop`: Warn
- `react-perf/jsx-no-new-function-as-prop`: Warn

#### Unused Imports/Vars

- `unused-imports/no-unused-imports`: Warn
- `unused-imports/no-unused-vars`: Warn, with options to ignore variables/args starting with `_` and ignore rest siblings.

### Settings

Configures shared settings for plugins:

- `react.version`: `detect` (auto-detects React version)
- `import/resolver`: Configures TypeScript and Node.js import resolution, with supported extensions.
- `import/extensions`: Lists allowed import file extensions.

### Overrides

Allows per-file or per-pattern rule adjustments. Examples:

- Disable explicit return types for all `.ts`/`.tsx` files.
- Use `espree` parser for `.js`/`.jsx` files.
- Allow CommonJS `require` in config files.
- Disable React performance warnings in components/hooks/contexts.
- Disable insecure randomness warning in utility files.

## Best Practices

1. **Rule Severity**: Use `"off"`/0 (disable), `"warn"`/1 (warning), or `"error"`/2 (error).
2. **VSCode Integration**: Install the ESLint extension for real-time feedback. Enable auto-fix on save:
   ```json
   // .vscode/settings.json
   {
     "editor.codeActionsOnSave": {
       "source.fixAll.eslint": true
     }
   }
   ```
3. **Custom Rules**: Adjust rule severity and add/remove rules to fit your team's workflow.
4. **Ignore Files**: Use `.eslintignore` or inline comments (`// eslint-disable-line`).
5. **Resolve Conflicts**: Place `prettier` last in `extends` and explicitly override conflicting rules.
6. **TypeScript & React**: Ensure `tsconfig.json` is correct and use React-specific rules for best practices.

```json
{
  "rules": {
    "react/react-in-jsx-scope": "off",
    "react/prop-types": "off",
    "react-hooks/rules-of-hooks": "error",
    "react-hooks/exhaustive-deps": "warn",
    "react/jsx-key": "error",
    "react/no-unescaped-entities": "warn",
    "react/display-name": "off"
  }
}
```

| 规则                            | 值        | 说明                                                   |
| ------------------------------- | --------- | ------------------------------------------------------ |
| `"react/react-in-jsx-scope"`    | `"off"`   | 不再需要在每个文件中导入 React（React 17+ 不需要）     |
| `"react/prop-types"`            | `"off"`   | 关闭 prop-types 检查，因为项目使用 TypeScript 类型     |
| `"react-hooks/rules-of-hooks"`  | `"error"` | 强制 React Hooks 规则，确保 Hooks 只在函数组件顶层调用 |
| `"react-hooks/exhaustive-deps"` | `"warn"`  | 检查 useEffect/useCallback 的依赖项列表，防止遗漏      |
| `"react/jsx-key"`               | `"error"` | 要求列表渲染时提供 key 属性                            |
| `"react/no-unescaped-entities"` | `"warn"`  | 警告 JSX 中未转义的特殊字符                            |
| `"react/display-name"`          | `"off"`   | 关闭组件显示名称检查                                   |

#### 无障碍访问规则

```json
{
  "rules": {
    "jsx-a11y/anchor-is-valid": "warn",
    "jsx-a11y/alt-text": "error",
    "jsx-a11y/img-redundant-alt": "error",
    "jsx-a11y/aria-role": "error"
  }
}
```

| 规则                           | 值        | 说明                                            |
| ------------------------------ | --------- | ----------------------------------------------- |
| `"jsx-a11y/anchor-is-valid"`   | `"warn"`  | 警告无效的 `<a>` 标签用法                       |
| `"jsx-a11y/alt-text"`          | `"error"` | 要求图片元素有 alt 属性                         |
| `"jsx-a11y/img-redundant-alt"` | `"error"` | 防止 alt 文本中包含多余的"image"、"picture"等词 |
| `"jsx-a11y/aria-role"`         | `"error"` | 要求 ARIA 角色属性值有效                        |

#### TypeScript 相关规则

```json
{
  "rules": {
    "@typescript-eslint/no-unused-vars": "off",
    "@typescript-eslint/no-explicit-any": "warn",
    "@typescript-eslint/explicit-function-return-type": "off",
    "@typescript-eslint/explicit-module-boundary-types": "off",
    "@typescript-eslint/no-non-null-assertion": "warn"
  }
}
```

| 规则                                                  | 值       | 说明                                             |
| ----------------------------------------------------- | -------- | ------------------------------------------------ |
| `"@typescript-eslint/no-unused-vars"`                 | `"off"`  | 关闭原始未使用变量检查，使用 unused-imports 替代 |
| `"@typescript-eslint/no-explicit-any"`                | `"warn"` | 警告使用 `any` 类型                              |
| `"@typescript-eslint/explicit-function-return-type"`  | `"off"`  | 不强制要求函数显式返回类型                       |
| `"@typescript-eslint/explicit-module-boundary-types"` | `"off"`  | 不强制要求导出函数和类的显式返回类型             |
| `"@typescript-eslint/no-non-null-assertion"`          | `"warn"` | 警告使用非空断言操作符 `!`                       |

#### 导入相关规则

```json
{
  "rules": {
    "import/order": [
      "warn",
      {
        "groups": [
          "builtin",
          "external",
          "internal",
          ["parent", "sibling"],
          "index",
          "object",
          "type"
        ],
        "newlines-between": "always",
        "alphabetize": {
          "order": "asc",
          "caseInsensitive": true
        }
      }
    ],
    "import/no-duplicates": "error",
    "import/no-unresolved": "off"
  }
}
```

| 规则                     | 值         | 说明                                                                                     |
| ------------------------ | ---------- | ---------------------------------------------------------------------------------------- |
| `"import/order"`         | 复杂配置   | 强制导入排序规则，按特定顺序分组并添加空行                                               |
| `"groups"`               | 数组       | 导入分组顺序：内置模块 → 外部依赖 → 内部模块 → 相对导入 → 索引文件 → 对象导入 → 类型导入 |
| `"newlines-between"`     | `"always"` | 不同组导入之间添加空行                                                                   |
| `"alphabetize"`          | 对象       | 按字母顺序排序导入，不区分大小写                                                         |
| `"import/no-duplicates"` | `"error"`  | 禁止重复导入同一模块                                                                     |
| `"import/no-unresolved"` | `"off"`    | 关闭未解析模块路径检查，TypeScript 已处理                                                |

#### 代码质量规则

```json
{
  "rules": {
    "no-console": ["warn", { "allow": ["warn", "error", "info"] }],
    "no-debugger": "warn",

    "sonarjs/cognitive-complexity": ["warn", 15],
    "sonarjs/no-duplicate-string": ["warn", { "threshold": 3 }],
    "sonarjs/no-identical-functions": "warn",

    "security-node/detect-insecure-randomness": "warn",

    "react-perf/jsx-no-new-object-as-prop": "warn",
    "react-perf/jsx-no-new-array-as-prop": "warn",
    "react-perf/jsx-no-new-function-as-prop": "warn"
  }
}
```

| 规则                                         | 值                             | 说明                                     |
| -------------------------------------------- | ------------------------------ | ---------------------------------------- |
| `"no-console"`                               | `["warn", { "allow": [...] }]` | 警告使用 console，但允许 warn/error/info |
| `"no-debugger"`                              | `"warn"`                       | 警告使用 debugger 语句                   |
| `"sonarjs/cognitive-complexity"`             | `["warn", 15]`                 | 限制函数认知复杂度不超过 15              |
| `"sonarjs/no-duplicate-string"`              | `["warn", { "threshold": 3 }]` | 警告重复字符串出现超过 3 次              |
| `"sonarjs/no-identical-functions"`           | `"warn"`                       | 警告完全相同的函数实现                   |
| `"security-node/detect-insecure-randomness"` | `"warn"`                       | 警告使用不安全的随机数生成               |
| `"react-perf/jsx-no-new-object-as-prop"`     | `"warn"`                       | 警告在 JSX 中内联创建对象（影响性能）    |
| `"react-perf/jsx-no-new-array-as-prop"`      | `"warn"`                       | 警告在 JSX 中内联创建数组（影响性能）    |
| `"react-perf/jsx-no-new-function-as-prop"`   | `"warn"`                       | 警告在 JSX 中内联创建函数（影响性能）    |

#### 未使用导入检查

```json
{
  "rules": {
    "unused-imports/no-unused-imports": "warn",
    "unused-imports/no-unused-vars": [
      "warn",
      {
        "vars": "all",
        "varsIgnorePattern": "^_",
        "args": "after-used",
        "argsIgnorePattern": "^_",
        "ignoreRestSiblings": true,
        "caughtErrors": "all",
        "caughtErrorsIgnorePattern": "^_"
      }
    ]
  }
}
```

| 规则                                 | 值       | 说明                                   |
| ------------------------------------ | -------- | -------------------------------------- |
| `"unused-imports/no-unused-imports"` | `"warn"` | 警告未使用的导入语句                   |
| `"unused-imports/no-unused-vars"`    | 复杂配置 | 警告未使用的变量                       |
| `"vars": "all"`                      |          | 检查所有变量                           |
| `"varsIgnorePattern": "^_"`          |          | 忽略以下划线开头的变量（如 `_unused`） |
| `"args": "after-used"`               |          | 只检查使用位置之后的参数               |
| `"argsIgnorePattern": "^_"`          |          | 忽略以下划线开头的参数                 |
| `"ignoreRestSiblings": true`         |          | 忽略解构赋值中的剩余部分               |
| `"caughtErrors": "all"`              |          | 检查所有 catch 子句错误参数            |
| `"caughtErrorsIgnorePattern": "^_"`  |          | 忽略以下划线开头的错误参数             |

### 设置配置

```json
{
  "settings": {
    "react": {
      "version": "detect"
    },
    "import/resolver": {
      "typescript": {
        "project": "./tsconfig.json"
      },
      "node": {
        "extensions": [".js", ".jsx", ".ts", ".tsx"]
      }
    },
    "import/extensions": [".js", ".jsx", ".ts", ".tsx"]
  }
}
```

`settings` 字段提供共享设置：

| 设置                  | 值         | 说明                                      |
| --------------------- | ---------- | ----------------------------------------- |
| `"react.version"`     | `"detect"` | 自动检测项目中使用的 React 版本           |
| `"import/resolver"`   | 对象       | 配置导入解析器                            |
| `"typescript"`        | 对象       | TypeScript 解析器配置，指向 tsconfig.json |
| `"node"`              | 对象       | Node.js 解析器配置，指定支持的文件扩展名  |
| `"import/extensions"` | 数组       | 指定导入时支持的文件扩展名                |

### 特定文件覆盖配置

```json
{
  "overrides": [
    {
      "files": ["*.tsx", "*.ts"],
      "rules": {
        "@typescript-eslint/explicit-function-return-type": "off"
      }
    },
    {
      "files": ["*.js", "*.jsx"],
      "parser": "espree",
      "parserOptions": {
        "ecmaVersion": 2022,
        "sourceType": "module",
        "ecmaFeatures": {
          "jsx": true
        }
      }
    },
    {
      "files": ["next.config.js", "tailwind.config.js", "postcss.config.js"],
      "rules": {
        "@typescript-eslint/no-var-requires": "off"
      }
    },
    {
      "files": [
        "**/contexts/**/*.tsx",
        "**/hooks/**/*.ts",
        "**/components/**/*.tsx",
        "**/app/**/*.tsx"
      ],
      "rules": {
        "react-perf/jsx-no-new-object-as-prop": "off",
        "react-perf/jsx-no-new-function-as-prop": "off",
        "react-perf/jsx-no-new-array-as-prop": "off"
      }
    },
    {
      "files": ["**/utils/**/*.ts"],
      "rules": {
        "security-node/detect-insecure-randomness": "off"
      }
    }
  ]
}
```

`overrides` 字段允许为特定文件指定不同的规则：

| 覆盖配置 | 目标文件          | 说明                                                                       |
| -------- | ----------------- | -------------------------------------------------------------------------- |
| 第1项    | `"*.tsx", "*.ts"` | 对所有 TypeScript 文件关闭显式函数返回类型要求                             |
| 第2项    | `"*.js", "*.jsx"` | 对 JavaScript 文件使用 espree 解析器                                       |
| 第3项    | 配置文件          | 对配置文件允许 CommonJS 的 require 语法                                    |
| 第4项    | 组件/Hooks/上下文 | 对组件、钩子和上下文文件关闭 React 性能警告（这些地方常需要内联函数/对象） |
| 第5项    | 工具函数          | 对工具函数关闭不安全随机数警告                                             |

## 最佳实践

1. **规则严重程度级别**：
   - `"off"` 或 `0`：完全禁用规则
   - `"warn"` 或 `1`：将规则视为警告（不影响退出码）
   - `"error"` 或 `2`：将规则视为错误（触发时退出码为1）

2. **VSCode 集成**：
   - 安装 ESLint 扩展以在编辑器中实时显示问题
   - 配置保存时自动修复：
     ```json
     // .vscode/settings.json
     {
       "editor.codeActionsOnSave": {
         "source.fixAll.eslint": true
       }
     }
     ```

3. **自定义规则**：
   - 根据项目需求调整规则严重程度
   - 添加或删除规则以适应开发团队习惯

4. **忽略特定文件**：
   - 使用 `.eslintignore` 文件或文件内的注释：`// eslint-disable-line`

5. **解决规则冲突**：
   - 多个扩展可能有冲突规则，通常应将 `"prettier"` 放在 `extends` 列表最后
   - 显式设置冲突规则的值以覆盖预设

6. **处理 TypeScript 和 React**：
   - 确保 `tsconfig.json` 路径正确
   - 使用 React 特定规则确保组件符合最佳实践
