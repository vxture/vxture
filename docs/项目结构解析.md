# 项目结构解析

本文档详细解析 Vxture 项目的目录结构，帮助开发者理解各个文件和目录的用途，以及它们之间的关系。

## 前端结构（`src/`）

```text
src/
├── app/                # Next.js App Router
│   ├── (features)/     # 按功能分组的路由
│   │   ├── auth/       # 认证相关页面
│   │   ├── dashboard/  # 控制面板页面
│   │   └── marketing/  # 营销相关页面
│   ├── api/            # API 路由（无内置聊天/智能体 API）
│   └── page.tsx        # 首页
├── components/         # 可复用组件
├── lib/                # 工具库
├── styles/             # 全局样式
└── types/              # TypeScript 类型定义
```

### `app/` 目录详解

Next.js 14+ 使用 App Router 架构，主要特点：

- 文件系统路由：目录结构决定 URL 路径
- 路由组 `(features)/`：使用括号命名目录用于逻辑分组，不影响 URL
- 特殊文件：`page.tsx` 定义路由页面，`layout.tsx` 定义布局

API 路由：`api/` 目录包含后端 API 端点，使用 `route.ts` 定义处理函数。

服务器组件：默认组件为 React 服务器组件（RSC），客户端组件需使用 `'use client'` 指令。

### `components/` 目录

组件按照职责与复用性划分为：

- `common/`：业务组件（示例：`UserProfile.tsx`，内置聊天组件已移除；如需聊天 UI，请通过外部服务集成）
- `layout/`：布局组件（例如 `Header.tsx`, `Footer.tsx`）
- `ui/`：UI 基础组件（例如 `Button.tsx`）

### `lib/` 目录

- `contexts/`：React 上下文（示例：`AuthContext.tsx`）
- `hooks/`：自定义 Hooks（示例：`useAuth.ts`，与聊天/智能体相关的 Hook 已移除）
- `utils/`：工具函数

## 后端结构（`backend/`）

```text
backend/
├── app/                # 应用代码
│   ├── api/            # API 端点
│   ├── core/           # 核心配置
│   ├── db/             # 数据库模型和连接
│   ├── models/         # 数据模型
│   ├── schemas/        # 数据验证模式
│   ├── services/       # 业务服务
│   └── utils/          # 工具函数
├── data/               # 数据文件
├── scripts/            # 实用脚本
└── tests/              # 测试代码
```

后端主要组织：

- `api/`：路由处理器与控制器
- `core/`：应用设置与配置管理
- `db/`：数据库连接与迁移
- `models/`：ORM 模型（示例：`User.py`）
- `schemas/`：Pydantic 验证模式（示例：`UserSchema.py`）

## 重要文件说明

- `package.json`：项目依赖与脚本

## 开发约定

- 组件文件使用 PascalCase（例如 `Button.tsx`）
- 工具函数使用 camelCase（例如 `formatDate.ts`）
- 导入顺序：外部库 → 内部模块 → 类型 → 样式

## 路由结构（示例）

- `/`：首页
- `/about`：关于
- `/dashboard`：用户控制面板

## 项目结构演进建议

- 新增特性时创建对应 `(features)/` 目录
- 随着组件库增长可考虑按设计系统（Atoms/Molecules/Organisms）组织
- 后端可按微服务拆分大型模块

## 结论

Vxture 项目采用现代化 Next.js + FastAPI 架构，目录清晰分离职责，遵循上述约定有助于提高代码可维护性。
