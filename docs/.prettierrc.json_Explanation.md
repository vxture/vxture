# .prettierrc.json Configuration Explained

This document explains the structure and options of the `.prettierrc.json` file in the Vxture project, focusing on modern Prettier usage and best practices for a TypeScript/JavaScript monorepo.

## What is .prettierrc.json?

`.prettierrc.json` is the configuration file for Prettier, a code formatter that enforces consistent code style across your project. Prettier automatically formats code according to defined rules, reducing style debates and improving team consistency.

## Example Configuration

```json
{
  "semi": true,
  "singleQuote": true,
  "tabWidth": 2,
  "trailingComma": "es5",
  "printWidth": 100,
  "bracketSpacing": true,
  "arrowParens": "avoid",
  "endOfLine": "lf"
}
```

## Option Breakdown

### Core Formatting Rules

- **semi**: `true` — Always add semicolons at the end of statements.
- **singleQuote**: `true` — Use single quotes instead of double quotes.
- **tabWidth**: `2` — Indent with 2 spaces.
- **printWidth**: `100` — Wrap lines at 100 characters.

### Commas, Brackets, and Arrows

- **trailingComma**: `'es5'` — Add trailing commas where valid in ES5 (objects, arrays, etc.).
- **bracketSpacing**: `true` — Add spaces inside object literals: `{ foo: bar }`.
- **arrowParens**: `'avoid'` — Omit parentheses for single-argument arrow functions: `x => x * x`.

### File Endings

- **endOfLine**: `'lf'` — Use LF (Line Feed) for line endings, recommended for cross-platform projects.

## Example: Before & After Formatting

**Before:**

```javascript
const user = { name: "John", age: 30, isAdmin: false };
function calculateTotal(items, tax) {
  return items.reduce((acc, item) => acc + item.price, 0) * (1 + tax);
}
const greeting = (name) => {
  return "Hello, " + name + "!";
};
```

**After Prettier:**

```javascript
const user = { name: "John", age: 30, isAdmin: false };

function calculateTotal(items, tax) {
  return items.reduce((acc, item) => acc + item.price, 0) * (1 + tax);
}

const greeting = (name) => {
  return "Hello, " + name + "!";
};
```

## Integrating with ESLint

Prettier handles formatting, while ESLint focuses on code quality. For best results:

1. Add Prettier to your ESLint config:

```json
{
  "extends": ["prettier"],
  "plugins": ["prettier"],
  "rules": {
    "prettier/prettier": "error"
  }
}
```

2. Install dependencies:

```bash
npm install --save-dev eslint-config-prettier eslint-plugin-prettier
```

## Editor Integration

For VS Code:

1. Install the Prettier extension.
2. Enable format on save:

```json
// .vscode/settings.json
{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true
}
```

## Best Practices

1. **Consistency**: Keep Prettier config consistent across the team to avoid unnecessary diffs.
2. **.prettierignore**: Use a `.prettierignore` file to exclude build output, generated files, dependencies, etc. Example:

```text
build/
dist/
.next/
node_modules/
*.min.js
```

3. **CI Integration**: Add a format check to your CI pipeline:

```json
// package.json
{
  "scripts": {
    "format:check": "prettier --check '**/*.{js,ts,tsx,json,md,scss}'"
  }
}
```

4. **Tool Compatibility**: Ensure Prettier config does not conflict with ESLint, TypeScript, etc.
5. **Document Exceptions**: If your team chooses non-default rules, document the reasons.

## Troubleshooting

1. **Formatting Conflicts**: If ESLint and Prettier conflict, make sure `eslint-config-prettier` is set up and `prettier` is last in `extends`.
2. **Files Not Formatting**: Check `.prettierignore`, file type support, and editor settings.
3. **Custom Formatting for File Types**: Use the `overrides` option in `.prettierrc.json`:

```json
{
  "overrides": [
    {
      "files": "*.md",
      "options": {
        "tabWidth": 4
      }
    }
  ]
}
```
