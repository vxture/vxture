# 数据库配置解析

本文档详细解析项目中的数据库配置方式，包括环境变量设置、连接方式以及主要使用的数据库技术。

## 基础数据库配置

项目使用环境变量管理数据库连接配置，主要配置项存储在 `.env.local` 文件中（本地开发环境）。同时提供了一个模板文件 `.env.local.template` 作为参考。

### 环境变量配置示例

```dotenv
# 数据库配置
DATABASE_URL=postgresql://postgres:password@localhost:5432/vxture
REDIS_URL=redis://localhost:6379/0
```

## 数据库技术选择

项目采用以下数据库技术：

1. **PostgreSQL** - 主数据库
   - 默认端口：5432
   - 使用 `pgvector` 扩展支持向量搜索（用于AI/RAG系统）
   - 连接字符串格式：`postgresql://用户名:密码@主机:端口/数据库名`

2. **Redis** - 缓存和会话管理
   - 默认端口：6379
   - 连接字符串格式：`redis://主机:端口/数据库索引`
   - 主要用途：缓存、会话存储、速率限制

## 数据库配置最佳实践

1. **环境变量分离**
   - 开发环境：`.env.local`（不提交到版本控制）
   - 测试环境：`.env.test`
   - 生产环境：在部署平台（如Vercel、Railway）中配置

2. **安全考虑**
   - 避免在代码中硬编码数据库凭据
   - 生产环境使用复杂密码
   - 限制数据库用户权限
   - 定期更换数据库密码

3. **连接池配置**
   - 后端使用连接池管理数据库连接
   - 根据应用负载调整连接池大小

## 数据库结构

项目中数据库结构遵循以下设计原则：

1. **模式组织**
   - 核心业务数据存储在主模式中
   - 向量数据可能存储在专用模式中
   - 安全相关表（如用户、权限）单独管理

2. **表设计**
   - 使用UUID作为主键
   - 所有表包含创建和更新时间戳
   - 实现软删除机制（deleted_at字段）
   - 使用外键保持数据完整性

## 部署配置

### 本地开发环境

本地开发环境通常使用Docker容器运行PostgreSQL和Redis：

```bash
# 启动PostgreSQL (含pgvector扩展)
docker run -d --name vxture-postgres -e POSTGRES_PASSWORD=password -p 5432:5432 ankane/pgvector

# 启动Redis
docker run -d --name vxture-redis -p 6379:6379 redis
```

### 生产环境

生产环境推荐使用托管数据库服务：

1. **PostgreSQL**:
   - AWS RDS for PostgreSQL（支持pgvector）
   - Railway PostgreSQL
   - Supabase PostgreSQL

2. **Redis**:
   - AWS ElastiCache
   - Upstash Redis
   - Railway Redis

## 相关代码示例

在实际开发中，数据库连接通常在后端服务中初始化，例如：

```python
# FastAPI后端示例 (backend/app/db/database.py)
from sqlalchemy import create_engine
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker
import os

# 从环境变量获取数据库URL
DATABASE_URL = os.getenv("DATABASE_URL")

# 创建SQLAlchemy引擎
engine = create_engine(DATABASE_URL)

# 创建会话工厂
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

# 创建基础模型类
Base = declarative_base()

# 获取数据库会话的依赖项
def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

## 向量数据库配置

对于向量搜索功能，项目使用PostgreSQL的pgvector扩展：

```sql
-- 初始化pgvector扩展
CREATE EXTENSION IF NOT EXISTS vector;

-- 创建向量表示例
CREATE TABLE document_embeddings (
    id UUID PRIMARY KEY,
    content TEXT NOT NULL,
    embedding vector(1536) NOT NULL, -- OpenAI嵌入维度
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 创建向量索引
CREATE INDEX ON document_embeddings USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);
```

## 数据库迁移管理

项目使用Alembic进行数据库模式管理和迁移：

```bash
# 创建迁移脚本
alembic revision --autogenerate -m "描述变更"

# 应用迁移
alembic upgrade head
```

## 注意事项

1. 不要将敏感的数据库凭据提交到版本控制系统
2. 始终使用环境变量或安全的密钥管理系统存储凭据
3. 在生产环境中定期备份数据库
4. 为不同环境使用不同的数据库实例
5. 监控数据库性能，并根据需要进行优化
