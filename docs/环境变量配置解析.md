# 环境变量配置详解

本文档详细解释了 Vxture 项目中环境变量的配置方式和各环境变量的作用，帮助开发者正确设置项目的环境配置。

## 环境变量文件概述

Next.js 项目中通常使用 `.env.*` 文件来管理环境变量。在 Vxture 项目中，我们使用以下文件：

- `.env.local.template` - 环境变量模板，包含所有必需的环境变量及说明
- `.env.local` - 本地开发环境变量（不提交到版本控制）

## .env.local.template 文件详解

```dotenv
# 开发环境变量配置

# 应用设置
APP_ENV=development
DEBUG=true
SECRET_KEY=your-secret-key-here

# 数据库配置
DATABASE_URL=postgresql://postgres:password@localhost:5432/vxture
REDIS_URL=redis://localhost:6379/0

# AI模型配置
# 阿里通义API
DASHSCOPE_API_KEY=your-dashscope-api-key

# DeepSeek
DEEPSEEK_API_KEY=your-deepseek-api-key

# 备选: OpenAI (如需使用)
# OPENAI_API_KEY=your-openai-api-key
# OPENAI_API_BASE=https://api.openai.com/v1
```

### 环境变量详解

#### 应用设置

| 变量名       | 示例值                 | 说明                                                                     |
| ------------ | ---------------------- | ------------------------------------------------------------------------ |
| `APP_ENV`    | `development`          | 应用环境标识，可选值：`development`、`testing`、`production`             |
| `DEBUG`      | `true`                 | 是否启用调试模式，生产环境应设为 `false`                                 |
| `SECRET_KEY` | `your-secret-key-here` | 应用密钥，用于加密会话数据和令牌，**必须保密且在生产环境中使用强随机值** |

#### 数据库配置

| 变量名         | 示例值                                                 | 说明                                                                             |
| -------------- | ------------------------------------------------------ | -------------------------------------------------------------------------------- |
| `DATABASE_URL` | `postgresql://postgres:password@localhost:5432/vxture` | PostgreSQL 数据库连接字符串，格式：`postgresql://用户名:密码@主机:端口/数据库名` |
| `REDIS_URL`    | `redis://localhost:6379/0`                             | Redis 连接字符串，用于缓存和会话存储，格式：`redis://主机:端口/数据库编号`       |

#### AI模型配置

| 变量名              | 示例值                      | 说明                                                    |
| ------------------- | --------------------------- | ------------------------------------------------------- |
| `DASHSCOPE_API_KEY` | `your-dashscope-api-key`    | 阿里通义大模型 API 密钥，用于访问阿里云的 AI 服务       |
| `DEEPSEEK_API_KEY`  | `your-deepseek-api-key`     | DeepSeek AI 模型 API 密钥，用于访问 DeepSeek 的 AI 服务 |
| `OPENAI_API_KEY`    | `your-openai-api-key`       | (备选) OpenAI API 密钥，用于访问 OpenAI 的服务          |
| `OPENAI_API_BASE`   | `https://api.openai.com/v1` | (备选) OpenAI API 基础 URL，可以配置为自定义端点        |

## 环境变量使用指南

### 设置本地环境变量

1. 复制模板文件创建 `.env.local`：

   ```bash
   copy .env.local.template .env.local
   ```

2. 编辑 `.env.local` 文件，填入实际的值。

3. 重启开发服务器使环境变量生效：
   ```bash
   npm run dev
   ```

### 在代码中访问环境变量

#### 前端（浏览器）访问

只有以 `NEXT_PUBLIC_` 前缀开头的环境变量可以在浏览器端访问：

```typescript
// 这将在构建时被替换为实际值
const apiUrl = process.env.NEXT_PUBLIC_API_URL;
```

#### 服务器端访问

所有环境变量均可在服务器端组件、API 路由和服务器操作中访问：

```typescript
// 服务器组件或 API 路由
const secretKey = process.env.SECRET_KEY;
const dbUrl = process.env.DATABASE_URL;
```

### 生产环境配置

在生产环境中，应使用部署平台（如 Vercel、Railway 等）提供的环境变量设置界面配置环境变量，而不是依赖 `.env` 文件。

## 环境变量最佳实践

1. **安全性**:
   - 永远不要提交包含实际密钥的 `.env.local` 文件到版本控制系统
   - 使用强随机值作为密钥和密码
   - 定期轮换密钥和密码

2. **命名约定**:
   - 使用大写字母和下划线命名环境变量（如 `DATABASE_URL`）
   - 只有需要在浏览器端访问的变量才使用 `NEXT_PUBLIC_` 前缀

3. **类型安全**:
   - 考虑使用 `zod` 或类似库验证环境变量：

   ```typescript
   // src/lib/env.ts
   import { z } from 'zod';

   const envSchema = z.object({
     DATABASE_URL: z.string().url(),
     REDIS_URL: z.string().url(),
     SECRET_KEY: z.string().min(32),
     // 其他环境变量...
   });

   export const env = envSchema.parse(process.env);
   ```

4. **不同环境的配置**:
   - 开发环境: `.env.development.local`
   - 测试环境: `.env.test.local`
   - 生产环境: 通过部署平台配置

5. **调试环境变量问题**:
   如果环境变量不生效，检查:
   - 文件名是否正确（`.env.local`）
   - 变量名是否正确（区分大小写）
   - 是否重启了开发服务器
