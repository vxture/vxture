# .stylelintrc.json 配置文件详解

本文档详细解释了 Vxture 项目中 `.stylelintrc.json` 文件的各个部分及其作用，帮助开发者理解 Stylelint 样式规范检查配置。

## 什么是 .stylelintrc.json？

`.stylelintrc.json` 是 Stylelint 的配置文件，用于定义 CSS/SCSS 代码的样式规范检查规则。Stylelint 是一个强大的样式文件 linter，可以帮助避免错误并强制执行团队中的样式约定。

## 配置文件概览

```json
{
  "extends": ["stylelint-config-standard-scss", "stylelint-config-prettier-scss"],
  "rules": {
    "selector-class-pattern": null,
    "scss/at-import-partial-extension": null,
    "scss/at-import-partial-extension-blacklist": null,
    "scss/dollar-variable-pattern": null,
    "media-feature-range-notation": null,
    "no-invalid-position-declaration": null,
    "scss/at-rule-no-unknown": [
      true,
      {
        "ignoreAtRules": ["tailwind", "apply", "variants", "responsive", "screen"]
      }
    ]
  }
}
```

## 配置选项详解

### 扩展配置

```json
{
  "extends": ["stylelint-config-standard-scss", "stylelint-config-prettier-scss"]
}
```

`extends` 字段允许继承预定义的配置集，Vxture 项目继承了以下配置：

| 扩展配置                           | 说明                                                                     |
| ---------------------------------- | ------------------------------------------------------------------------ |
| `"stylelint-config-standard-scss"` | SCSS 的标准配置，基于 SCSS 语法和最佳实践提供了一组合理的规则            |
| `"stylelint-config-prettier-scss"` | 专为与 Prettier 配合使用设计的 SCSS 配置，禁用可能与 Prettier 冲突的规则 |

### 规则配置

```json
{
  "rules": {
    "selector-class-pattern": null,
    "scss/at-import-partial-extension": null,
    "scss/at-import-partial-extension-blacklist": null,
    "scss/dollar-variable-pattern": null,
    "media-feature-range-notation": null,
    "no-invalid-position-declaration": null,
    "scss/at-rule-no-unknown": [
      true,
      {
        "ignoreAtRules": ["tailwind", "apply", "variants", "responsive", "screen"]
      }
    ]
  }
}
```

#### 禁用的规则

| 规则                                           | 值     | 说明                                           |
| ---------------------------------------------- | ------ | ---------------------------------------------- |
| `"selector-class-pattern"`                     | `null` | 禁用类选择器命名模式检查，允许任意的类名格式   |
| `"scss/at-import-partial-extension"`           | `null` | 禁用 `@import` 中局部文件扩展名检查            |
| `"scss/at-import-partial-extension-blacklist"` | `null` | 禁用 `@import` 中局部文件扩展名黑名单检查      |
| `"scss/dollar-variable-pattern"`               | `null` | 禁用 SCSS 变量命名模式检查，允许任意变量名格式 |
| `"media-feature-range-notation"`               | `null` | 禁用媒体查询范围表示法检查                     |
| `"no-invalid-position-declaration"`            | `null` | 禁用在声明中使用无效位置属性的检查             |

#### 自定义规则

```json
{
  "scss/at-rule-no-unknown": [
    true,
    {
      "ignoreAtRules": ["tailwind", "apply", "variants", "responsive", "screen"]
    }
  ]
}
```

| 规则                        | 值              | 说明                                                |
| --------------------------- | --------------- | --------------------------------------------------- |
| `"scss/at-rule-no-unknown"` | `[true, {...}]` | 检查未知的 `@` 规则，但忽略 Tailwind CSS 特定的指令 |
| `"ignoreAtRules"`           | 数组            | 指定要忽略的 `@` 规则名称列表                       |

这个配置专门为 Tailwind CSS 添加了例外，允许使用以下 Tailwind 特定的指令：

- `@tailwind` - 引入 Tailwind 的基础样式、组件和工具类
- `@apply` - 在自定义 CSS 中应用 Tailwind 工具类
- `@variants` - 为多个变体生成组合规则
- `@responsive` - 生成响应式变体
- `@screen` - 创建媒体查询，使用 Tailwind 断点名称

## 与 Tailwind CSS 集成

该配置专门适配了 Tailwind CSS，确保 Stylelint 不会错误地将 Tailwind 指令报告为错误。这对于以下场景特别重要：

```scss
/* 使用 Tailwind 指令的 SCSS 示例 */
.custom-button {
  @apply bg-blue-500 text-white py-2 px-4 rounded;

  &:hover {
    @apply bg-blue-700;
  }

  @screen md {
    @apply py-3 px-6;
  }
}
```

## 与其他工具的集成

### 与 Prettier 集成

通过使用 `stylelint-config-prettier-scss` 配置，确保 Stylelint 规则不会与 Prettier 的格式化冲突。

### 与 VS Code 集成

1. 安装 Stylelint 扩展

2. 配置 VS Code 设置：
   ```json
   // .vscode/settings.json
   {
     "stylelint.validate": ["css", "scss"],
     "editor.codeActionsOnSave": {
       "source.fixAll.stylelint": true
     }
   }
   ```

## 使用示例

### 有效的 SCSS 代码示例

```scss
// 符合配置规则的 SCSS 代码
.feature-card {
  display: flex;
  margin-bottom: 2rem;

  &__icon {
    width: 48px;
    height: 48px;
    margin-right: 1rem;
  }

  &__content {
    flex: 1;
  }

  // 使用 Tailwind 指令
  &__title {
    @apply text-xl font-bold mb-2;
    color: $primary-color;
  }

  // 响应式设计
  @media (max-width: 768px) {
    flex-direction: column;

    &__icon {
      margin-bottom: 1rem;
      margin-right: 0;
    }
  }

  // 使用 Tailwind 断点
  @screen lg {
    margin-bottom: 3rem;
  }
}
```

### 常见错误及修复

| 错误            | 修复方法                            |
| --------------- | ----------------------------------- |
| 重复的选择器    | 合并或重构重复的选择器              |
| 无效的 CSS 属性 | 修正拼写错误或使用有效的 CSS 属性   |
| 颜色格式不一致  | 统一使用十六进制、RGB 或 HSL 格式   |
| 单位使用不一致  | 统一使用相同的单位系统（rem/em/px） |

## 最佳实践

1. **配置 Stylelint 脚本**：在 `package.json` 中添加 Stylelint 脚本：

   ```json
   {
     "scripts": {
       "lint:style": "stylelint \"src/**/*.scss\" --fix"
     }
   }
   ```

2. **创建 `.stylelintignore` 文件**：排除不需要检查的文件：

   ```
   node_modules/
   build/
   .next/
   public/
   ```

3. **与 CI/CD 集成**：在持续集成流程中添加样式检查步骤。

4. **保持配置与项目需求同步**：随着项目发展，定期审查和更新 Stylelint 配置。

5. **团队一致性**：确保所有团队成员使用相同的配置，提高代码审查效率。

## 常见问题解决

1. **与 Tailwind CSS 冲突**：
   - 确保 `scss/at-rule-no-unknown` 规则中包含所有 Tailwind 指令
   - 考虑使用 `stylelint-config-recommended-tailwindcss` 配置

2. **与 Prettier 冲突**：
   - 确保使用 `stylelint-config-prettier-scss` 配置
   - 运行 `npx stylelint-config-prettier-check` 检查冲突

3. **性能优化**：
   - 使用 `.stylelintignore` 排除不需要检查的文件
   - 考虑使用缓存选项：`--cache` 和 `--cache-location`
