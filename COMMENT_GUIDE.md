# COMMENT_GUIDE.md - Vxture 项目注释规范

本文档专门定义项目代码注释的详细规范，用于指导 AI 完善代码注释。

---

## 一、注释通用原则

### 1.1 基本要求

- **注释解释"为什么"而不是"是什么"** - 代码本身已经说明"做什么"
- **禁止无用注释** - 不要写 `// 增加1` 这样的废话
- **保持更新** - 代码修改时同步更新注释
- **中文注释** - 项目统一使用中文注释
- **包信息** - 所有文件头部必须添加 `@package` 标识所属包

### 1.2 包名说明

| 包名                    | 说明           |
| ----------------------- | -------------- |
| `@vxture/website`       | 官网应用       |
| `@vxture/design-system` | 设计系统组件库 |
| `@vxture/api`           | 后端 API 服务  |

### 1.3 注释类型

| 类型      | 用途               | 格式                      |
| --------- | ------------------ | ------------------------- |
| 文件头部  | 文件说明、版权信息 | JSDoc 块注释 `/** ... */` |
| 分区注释  | 代码块分隔         | `// ========` 格式        |
| 接口/类型 | 数据结构说明       | JSDoc 块注释              |
| 函数说明  | 功能、参数、返回值 | JSDoc 块注释              |
| 行内注释  | 复杂逻辑说明       | `//` 单行注释             |

## 二、文件头部注释（统一格式）

### 2.1 完整头部注释

**适用场景：** 核心业务文件、重要组件、Hooks、服务类

```typescript
/**
 * filename.ts - 简短描述
 * @package @vxture/website
 *
 * 功能：详细描述文件的功能和职责
 *
 * @author Stone Smoker
 * @created 2024-06-01
 * @lastModified 2026-03-05
 * @version 2.0.0
 * @copyright Copyright (c) 2024-2026 Vxture Team
 * @license MIT
 *
 * @layer Presentation
 * @category Components - User
 */
```

### 2.2 简化头部注释

**适用场景：** 简单组件、工具函数、类型定义、辅助文件

```typescript
/**
 * filename.ts - 简短描述
 * @package @vxture/design-system
 *
 * @copyright Vxture Team
 * @license MIT
 * @layer Presentation
 * @category Components - Common
 */
```

### 2.3 字段说明

| 字段           | 说明                     | 是否必填 |
| -------------- | ------------------------ | -------- |
| `filename.ts` | 文件名 + 一句话描述      | **是**   |
| `@package`     | 所属包名                 | **是**   |
| `功能：`       | 详细功能描述             | 完整版本必填 |
| `@author`      | 作者                     | 可选     |
| `@created`     | 创建日期                 | 可选     |
| `@lastModified`| 最后修改日期             | 可选     |
| `@version`     | 版本号                   | 可选     |
| `@copyright`   | 版权信息                 | **是**   |
| `@license`     | 许可证                   | **是**   |
| `@layer`       | 架构分层                 | **是**   |
| `@category`    | 分类                     | **是**   |

### 2.4 @layer 架构分层取值

| 取值             | 说明                              |
| ---------------- | --------------------------------- |
| `Presentation`   | 表示层（组件、页面、UI）          |
| `Application`    | 应用层（业务逻辑、Hooks、服务）   |
| `Domain`         | 领域层（类型、模型、规则）        |
| `Infrastructure` | 基础设施层（API、存储、外部服务） |
| `Shared`         | 共享层（工具、常量）              |

## 三、通用注释规范

### 3.1 分区注释

文件内部按逻辑划分代码块：

```typescript
// ============================================================================
// 类型定义
// ============================================================================

// ... 类型定义代码 ...

// ============================================================================
// 常量定义
// ============================================================================

// ... 常量定义代码 ...

// ============================================================================
// 组件/Hook 实现
// ============================================================================

// ... 主要业务代码 ...
```

### 3.2 接口/类型注释

```typescript
/**
 * 登录表单组件 Props 接口
 *
 * 用于接收登录表单所需的外部回调函数
 */
interface LoginFormProps {
  /** 登录成功后的回调函数 */
  readonly onSuccess?: () => void;

  /** 用户取消登录时的回调函数 */
  readonly onCancel?: () => void;

  /** 自定义样式类名 */
  readonly className?: string;
}
```

### 3.3 函数注释

```typescript
/**
 * 格式化日期为字符串
 *
 * @param date - 要格式化的日期对象或时间戳
 * @param options - 格式化选项
 * @returns 格式化后的日期字符串
 * @example
 * ```typescript
 * formatDate(new Date(), { includeTime: true });
 * // 返回: "2024-03-05 14:30:00"
 * ```
 */
export function formatDate(date: Date | number, options?: DateFormatOptions): string {
  // 实现...
}
```

### 3.4 枚举注释

```typescript
/**
 * 用户状态枚举
 */
export enum EUserStatus {
  /** 账户未激活 */
  INACTIVE = 'inactive',
  /** 账户正常激活 */
  ACTIVE = 'active',
  /** 账户被禁用 */
  DISABLED = 'disabled',
  /** 账户已注销 */
  DELETED = 'deleted'
}
```

## 四、检查清单

在完成代码注释后，请检查以下项目：

- [ ] 有文件头部注释
- [ ] **包含 `@package` 包信息**
- [ ] 包含 `@copyright` 和 `@license`
- [ ] 包含 `@layer` 和 `@category`
- [ ] 类型/接口有说明
- [ ] 函数有 JSDoc 注释（含参数、返回值）
- [ ] 复杂逻辑有行内注释
