# 视频资源使用说明

## 📁 视频文件位置

您的视频文件应该放在：

```
packages/web/public/videos/banner-hero-01.mp4
```

## 🎬 视频要求

### 技术规格

- **格式**: MP4 (推荐 H.264 编码)
- **分辨率**: 推荐 1920x1080 或更高
- **长宽比**: 16:9 (标准宽屏)
- **时长**: 建议 10-30 秒循环播放
- **文件大小**: 建议控制在 10MB 以内

### 内容建议

- **主题**: 数据流动、科技感、现代化办公场景
- **色调**: 蓝色、青色系，与当前设计保持一致
- **动态**: 缓慢、稳定的移动，避免快速闪烁
- **无音频**: 背景视频会自动静音播放

## 🛠️ 实现特性

### 1. 响应式适配

- **满屏显示**: 视频会自动填满整个可视区域
- **居中裁剪**: 使用 `object-cover` 保持比例，超出部分裁剪
- **多设备兼容**: 支持桌面、平板、手机设备

### 2. 加载优化

- **预加载**: 使用 `preload="metadata"` 提前加载视频信息
- **优雅降级**: 视频加载失败时自动显示备用背景
- **平滑过渡**: 1秒淡入淡出效果

### 3. 用户体验

- **自动播放**: 页面加载时自动开始播放
- **循环播放**: 视频结束后自动重新开始
- **静音播放**: 避免突然的音频干扰
- **移动端优化**: 使用 `playsInline` 属性避免全屏播放

## 🔧 代码实现

视频背景已集成到 HeroSection 组件中：

```tsx
<video
  ref={videoRef}
  className="absolute inset-0 w-full h-full object-cover"
  autoPlay
  muted
  loop
  playsInline
  preload="metadata"
>
  <source src="/videos/banner-hero-01.mp4" type="video/mp4" />
</video>
```

## 🎨 视觉效果

### 遮罩层

- 添加了渐变遮罩确保文字可读性
- 保持原有的深蓝色主题

### 备用背景

- 视频未加载时显示原有的渐变背景和动画
- 无缝切换，用户体验流畅

## 📱 兼容性

- **现代浏览器**: Chrome, Firefox, Safari, Edge
- **移动设备**: iOS Safari, Android Chrome
- **备用方案**: 不支持的浏览器会显示静态背景

## 🚀 使用步骤

1. 将您的 `banner-hero-01.mp4` 文件放入 `packages/web/public/videos/` 目录
2. 确保视频文件符合技术规格要求
3. 重新加载页面即可看到视频背景效果
