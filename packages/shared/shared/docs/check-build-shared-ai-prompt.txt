# Vxture AI Prompt: Shared Layer Check & Build

## Context
- Target Package: @vxture/shared
- Location: packages/shared/src
- All TypeScript types, constants, and pure utility functions reside here
- Shared layer MUST NOT include:
  - UI components, hooks, pages
  - Business logic
  - Dependencies on core, service, or portal packages
- Must follow architecture documentation:
  - docs/architecture/package-graph.json
  - docs/architecture/shared-layer.md
  - docs/architecture/package-boundaries.md
  - docs/ai/claude-coding-comments.md
  - docs/ai/claude-coding-rules.md
  - docs/ai/claude-coding-style.md
  - docs/ai/ai-context.md

## Tasks

### 1. Boundary Verification
- Ensure all types in `types/` are framework-agnostic and not page or UI-specific
- Constants in `constants/` must be generic and reusable
- Utilities in `utils/` must be pure functions only
- No references to:
  - Core layer (@vxture/core-*)
  - Service layer (@vxture/service-*)
  - UI/Portal layer (@vxture/ui-kit, portals)

### 2. TypeScript Check
- Verify `tsconfig.json` supports composite projects and path alias: `@vxture/shared/*`
- All TypeScript files must compile without errors
- Remove any duplicate or unnecessary TS utility types (Partial, Pick, Omit, Required)
- Shared types must have `Shared` prefix if needed to avoid conflicts (e.g., `SharedAPIResponse`)

### 3. File Header & Comments
- All TS files must follow standardized file header:
```ts
/**
 * filename.ts - Short description
 * @package @vxture/shared
 *
 * Description: Detailed explanation of the file's functionality and responsibilities
 *
 * @author AI-Generated
 * @date ${DATE} ${TIME}
 * @version 1.0
 *
 * @copyright Vxture Team
 * @license MIT
 *
 * @layer Shared
 * @category Components - Common
 */
```
- Maintain section headers and Claude-style comments

### 4. Index & Exports
- Ensure `index.ts` exports everything:
```ts
export * from './types/content.types';
export * from './constants';
export * from './utils';
```
- Provide example imports:
```ts
import { SharedAPIResponse, ContentTheme, Link } from '@vxture/shared';
```

### 5. Build & Output
- Compile the shared package
- Report any type errors, dependency violations, or misclassified types
- Output a report:
  - ✅ All types are valid and framework-agnostic
  - ✅ No forbidden dependencies
  - ✅ Index exports complete
  - ⚠️ Any issues found with location, naming, or boundaries

## Output
- Clean, copy-paste ready TypeScript code for shared package
- Complete `index.ts` export
- Report summary of checks and fixes applied
