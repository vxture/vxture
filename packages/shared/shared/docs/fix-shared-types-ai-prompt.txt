4. Follow **Claude coding style** and **comments standard**:
   - Use `@package`, `@layer`, `@category` in file headers
   - Section comments for grouping types
   - Explain "why", not "what"
   - Keep comments updated and consistent

5. Provide **example usage** for Core, Service, and Platform SDK layers:
   - Show how shared types and constants are imported and used
