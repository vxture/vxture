# Vxture AI Prompt – Phase 3: Core Layer

## Context
- Target Packages: @vxture/core-* (all core packages)
- Location: packages/core-*/src
- Core layer packages provide **platform infrastructure**, including:
  - API handling
  - Tenant & locale management
  - Authentication & authorization utilities
  - Core services used by other layers (Shared, Service, Platform SDK)
- Must follow architecture documentation:
  - docs/architecture/core-layer.md
  - docs/architecture/package-boundaries.md
  - docs/architecture/package-graph.json
  - docs/ai/claude-coding-comments.md
  - docs/ai/claude-coding-rules.md
  - docs/ai/claude-coding-style.md
  - docs/ai/ai-context.md
- Dependencies allowed:
  - `@vxture/shared`
  - Only third-party libraries (npm)
- Forbidden dependencies:
  - services/*
  - portals/UI packages
  - Platform SDK packages

## Objective

1. **Refactor and implement core packages (`core-*`)**:
   - Consolidate existing Core code into modular, reusable packages
   - Ensure **layer correctness**:
     - Core layer may depend on Shared and allowed third-party libs
     - Must **not** depend on Service or Portal/UI packages
   - Ensure TypeScript composite configuration works
   - Maintain path aliases: `@vxture/core/*`

2. **Enforce strict boundaries and modularity**:
   - Each package should have clear responsibilities
   - Avoid duplication of shared utilities or types (use `@vxture/shared`)
   - Ensure API, tenant, locale, and auth utilities are properly typed and exported

3. **Follow Claude coding standards**:
   - File headers with `@package`, `@layer`, `@category`, etc.
   - Section comments for grouping related types and functions
   - Explain "why", not "what"
   - Keep code consistent and maintainable
   - Use descriptive TypeScript types, interfaces, and utility types

4. **Provide example usage for dependent layers**:
   - Service packages consuming Core APIs
   - Platform SDK consuming Core utilities
   - Show **import statements** and minimal usage examples

5. **Export and index files**:
   - Each core package must have `index.ts` exporting public API
   - Ensure consistent naming and path alias usage
