# Vxture AI Prompt: Phase 3 Core Layer

## Context
- Target Packages: @vxture/core-*
- Location: packages/core-*
- Core layer provides **platform infrastructure**, utilities, and services
- Core MUST NOT include:
  - Business logic
  - UI components or hooks
  - Page-specific logic
  - Dependencies on Service, Portal, or Platform SDK packages
- Must follow architecture and coding standards:
  - docs/architecture/core-layer.md
  - docs/architecture/package-boundaries.md
  - docs/architecture/package-graph.json
  - docs/ai/claude-coding-comments.md
  - docs/ai/claude-coding-rules.md
  - docs/ai/claude-coding-style.md
  - docs/standards/vx-TsconfigConfig.md
  - docs/ai/ai-context.md

## Objective
1. **Build/Refactor Core Packages**:
   - core-api: centralized API infrastructure
   - core-tenant: tenant context and multi-tenant utilities
   - core-locale: locale and i18n tools
   - core-auth: authentication and authorization helpers
   - core-utils: general platform utilities

2. **Maintain Layer Correctness**:
   - Core can depend only on `@vxture/shared`
   - Core must be framework-agnostic (Node.js + frontend)
   - No React/Next.js/DOM-specific code

3. **Ensure High-Quality TypeScript**:
   - Use composite configuration for buildable packages
   - Strict typing enforced
   - Maintain path aliases: `@vxture/core-*/*`
   - Export all public symbols via `index.ts`

4. **Follow Claude Coding Style & Comments Standard**:
   - File headers must include `@package`, `@layer`, `@category`
   - Use section comments to group related functionality
   - Explain "why", not "what"
   - Consistent annotation across all core packages

5. **Provide Example Usage for Service and Platform SDK layers**:
   - Demonstrate imports from core packages
   - Illustrate combined usage of core-api, core-tenant, core-locale, core-auth, core-utils

## Constraints
- Only modify `packages/core-*` packages
- Do **not** touch shared, service, portal, or platform SDK packages
- Do **not** delete or move prompt or documentation files
- Respect architecture boundaries in `package-boundaries.md` and `core-layer.md`
- Forbidden dependencies: service, portal, platform SDK
- Keep Core reusable, composable, and framework-agnostic

## Output Requirements
1. Full TypeScript implementation for all core packages:
   - `core-api`
   - `core-tenant`
   - `core-locale`
   - `core-auth`
   - `core-utils`
2. Centralized `index.ts` for each package exporting all public APIs
3. Example import statements demonstrating usage in:
   - Service layer
   - Platform SDK layer
   - Portal layer (if needed)
4. All code **copy-paste ready**, fully compliant with Vxture Claude coding standards
5. Composite tsconfig.json ready for buildable packages
