# CODE_STYLE.md - Vxture 项目代码规范

---

## 一、编码规范

### 1.1 通用原则

- 可读性优先：清晰 &gt; 简洁 &gt; 聪明
- 单一职责：一个函数/组件只做一件事
- 高内聚低耦合：模块内部紧密关联，模块之间松耦合
- 类型安全：充分利用 TypeScript 类型系统，杜绝无约束类型使用
- 禁止硬编码：固定值必须引用对应常量
- 早期返回：减少嵌套层级，提升代码可读性
- 提取函数：复杂逻辑拆分提取，降低认知复杂度
- 分层架构：遵循项目分层架构，禁止跨层调用

### 1.2 格式规范

- 缩进：前端 2 空格，后端 4 空格
- 行宽：不超过 120 字符
- 文件编码：UTF-8
- 结尾换行：必须保留
- 引号：前端统一使用单引号 '，JSX 使用双引号 "；后端统一使用单引号 '

### 1.3 命名规范

- 组件：PascalCase 大驼峰
- 页面文件：kebab-case 短横线
- 工具函数、变量：camelCase 小驼峰
- 常量：全大写 + 下划线（UPPER_SNAKE_CASE）
- 接口/类型：以 I 开头 + PascalCase
- 枚举：以 E 开头 + PascalCase
- 后端函数/变量：snake_case 下划线
- 后端类名：PascalCase 大驼峰
- 路由名称：kebab-case

### 1.4 React 规范

- 只使用函数组件 + Hooks，禁止使用 class 组件
- Props 必须定义接口，禁止 any 类型
- Props 接口的所有属性标记为 readonly
- 状态提升合理，避免过度 props drilling
- 服务端组件 / 客户端组件明确区分
- 客户端组件必须添加 'use client'

### 1.5 TypeScript 规范

- 禁止使用 any、unknown 无约束使用
- 必须开启严格类型校验
- 接口优先于 type 定义对象结构
- 类型复用优先使用泛型
- 禁止隐式 any
- 优先使用可选链（?.）简化判断逻辑

### 1.6 TailwindCSS 规范

- 只使用 TailwindCSS 4，禁止原生 CSS
- 类名按逻辑排序：布局 → 尺寸 → 边框 → 文字 → 交互
- 复用样式使用工具类组合，禁止重复代码
- 禁止内联样式

### 1.8 后端规范（FastAPI + Python）

- 严格遵循 PEP8 规范
- 所有接口必须定义请求/响应模型
- 路由层 → 服务层 → 数据层，禁止在路由中写核心逻辑

### 1.9 静态数据与常量

**静态数据（/public/data 目录）**

- 文件命名：kebab-case 短横线命名
- 按功能模块拆分子目录
- JSON 文件需格式化（缩进 2 空格）
- 读取时直接使用 Next.js 公共目录路径（/data/xxx/xxx.json）

**静态变量**

- 命名：全大写 + 下划线（UPPER_SNAKE_CASE）
- 存放位置：统一放在 shared/constants/ 目录下
- 按模块拆分文件
- 必须结合 TypeScript 类型定义

## 二、注释规范

### 2.1 通用原则

- 注释解释"为什么"而不是"是什么" - 代码本身已经说明"做什么"
- 禁止无用注释 - 不要写 `// 增加1` 这样的废话
- 保持更新 - 代码修改时同步更新注释
- 中文注释 - 项目统一使用中文注释
- 包信息 - 所有文件头部必须添加 `@package` 标识所属包

### 2.3 文件头部注释

**核心文件**

```typescript
/**
 * filename.ts - 简短描述
 * @package @vxture/[app-name]
 *
 * 功能：详细描述文件的功能和职责
 *
 * @copyright Copyright (c) 2024-2026 Vxture Team
 * @license MIT
 *
 * @layer Presentation
 * @category Components - Common
 */
```

### 2.4 @layer 架构分层取值

| 取值             | 说明                              |
| ---------------- | --------------------------------- |
| `Presentation`   | 表示层（组件、页面、UI）          |
| `Application`    | 应用层（应用逻辑、Hooks、服务）   |
| `Domain`         | 领域层（类型、模型、规则）        |
| `Infrastructure` | 基础设施层（API、存储、外部服务） |
| `Shared`         | 共享层（工具、常量）              |

### 2.5 分区注释

```typescript
// ============================================================================
// 类型定义
// ============================================================================

// ... 类型定义代码 ...

// ============================================================================
// 常量定义
// ============================================================================

// ... 常量定义代码 ...

// ============================================================================
// 组件/Hook 实现
// ============================================================================

// ... 主要代码 ...
```

### 2.6 接口/类型注释

```typescript
/**
 * 接口说明
 */
interface ExampleProps {
  /** 属性说明 */
  readonly property?: Type;
}
```

### 2.7 函数注释

```typescript
/**
 * 函数说明
 * @param param - 参数说明
 * @returns 返回值说明
 */
export function functionName(param: Type): ReturnType {
  // 实现...
}
```

### 2.8 枚举注释

```typescript
/**
 * 枚举说明
 */
export enum EExample {
  /** 成员说明 */
  MEMBER = 'value',
}
```

### 2.9 后端函数 docstring（Python）

```python
def function_name(param: str) -&gt; Optional[ReturnType]:
    """
    函数说明
    Args:
        param: 参数说明
    Returns:
        返回值说明
    Raises:
        异常说明
    """
    # 实现...
```
