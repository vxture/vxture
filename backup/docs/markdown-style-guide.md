# Markdown 规范说明

本项目使用 markdownlint 来保证 Markdown 文档的规范性和一致性。以下是主要规则说明：

## 标题规则

- **MD001**: 标题层级不应该跳跃（例如，H1 后面直接使用 H3）
- **MD003**: 使用 ATX 风格的标题（使用 `#` 符号）
- **MD022**: 标题前后应有空行
- **MD023**: 标题前不应有空格
- **MD024**: 不同嵌套层级下可以有相同的标题
- **MD025**: 文档中应只有一个顶级标题（# 一级标题）
- **MD026**: 标题中避免使用标点符号

## 列表规则

- **MD007**: 无序列表缩进为 2 个空格
- **MD029**: 有序列表使用顺序编号（1. 2. 3.）
- **MD030**: 列表标记后应有空格

## 空格和格式

- **MD009**: 行尾空格，仅在需要换行时使用 2 个空格
- **MD010**: 代码块以外禁止使用硬 Tab
- **MD012**: 避免多个连续空行
- **MD013**: 不限制行长度
- **MD037**: 强调标记（如 _斜体_）内不应有多余空格

## 代码块

- **MD040**: 代码块应指定语言
- **MD046**: 使用围栏式代码块（使用 ``` 标记）

## 其他规则

- **MD033**: 允许使用内联 HTML
- **MD034**: 避免使用裸 URL，应该使用链接格式 `[文本](URL)`
- **MD036**: 允许使用强调作为标题
- **MD041**: 不要求文件开头必须是顶级标题

## VS Code 集成

若使用 VS Code 编辑器，请安装 `davidanson.vscode-markdownlint` 插件以获得实时提示和自动修复功能。
